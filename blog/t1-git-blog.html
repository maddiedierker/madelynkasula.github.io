Adventures in Version Control

Git and GitHub

14 June 2015

What are the benefits of version control? Version control allows you to track progress, makes collaboration easier, rollback to previous versions if needed (i.e., a bug was introduced, a feature was added that is no longer wanted, etc.), and save your skin if you somehow delete/ruin/burn your current progress by backing up your work. Without version control, you would be stuck if you had accidentally broken something in your code because it would be much more complicated to undo what you (or someone else) had done. Along with that, collaboration would be confusing as well--if the feature you're working on somehow affects someone else's feature that they just finished, you would have much more trouble rectifying the problems if you didn't have the most current version. Your code would work on the project you have, but if the new feature isn't incorporated, you're going to be going back and debugging for who knows how long.

How does git help you keep track of changes? You can keep track of changes by making commits as often as possible--create a feature branch for your work, merge it, and commit to git whenever you can. For example, when answering reflection questions for homework on git repositories, I created a branch called repo-reflection to start working on these questions. Once I answer the questions in my text file, I can commit these changes, merge my repo-reflection branch back into the main (master) branch, and delete the no-longer-needed feature branch. Now that I have changed my project, I can use git log to look at a nice synopsis of my work. This is why commit messages are so important--when I look at my git log, it shows each commit, which should be able to quickly tell me the history of my project. When your commit messages are good, you can track all changes on your project using this log to see your progress! Lesson: Write good commit messages. Or else.

Why use GitHub to store your code? GitHub is used so widely by developers and software companies because of its ease and usefulness. Storing your code on GitHub allows you to back up your projects easily, display your code for others to use/see (and vice versa), and makes virtual collaboration much easier! You can find open source projects to contribute to, work on your own projects with someone 700 miles away very easily by viewing the repository as a whole, and show your chops to companies looking to hire you. Now go ahead and back that code up.
