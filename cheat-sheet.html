Three Great Git/GitHub Recipes

recipe #1: make new local repo & upstream

    - Move to source repo page on GitHub
    - Use Fork button
    - Change new repo setting to allow Issues
    - Copy repo URL
    - Change to local directory for repo(s)
    - git clone [URL]
    - Snag url from original, for upstream
    - git remote add upstream [URL]
    - To check: git remote -v

recipe #2: fetch upstream changes

    - Shift to the origin directory
    - Check using git remote -v
    - git fetch upstream
    - git branch -v will show upstream/master
    - Shift to master branch
    - git merge upstream/master (or merge any [remote]/[branch])
    - May need to commit
    - Conclude with git push origin master

recipe #3: push task(s)

    - Use git add . (adds all tracked files... use [filename] for specific files) and git commit -m "Your commit description" to commit
    - git pull to pull changes before pushing new changes
    - git push origin [branch] to push changes to Github
    - checkout master
    - git merge [branch] to fastforward the master
